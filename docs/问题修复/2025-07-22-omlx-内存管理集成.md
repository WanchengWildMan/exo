# 2025-07-22 oMLX内存管理集成（P0/P1/P2）

## 背景

从 [oMLX](https://github.com/jundot/omlx)（8.5k stars）学习其内存管理策略，融合到 exo 的 MLX 推理引擎中。
主要目标：解决 2×16GB Mac Mini（M2+M4）分布式推理 Qwen3.5-9B 时的 OOM 问题。

## 变更概览

### P0-A: mx.synchronize() + mx.clear_cache() 保护
**问题**: oMLX 发现 `mx.clear_cache()` 在未同步的情况下可能触发 M4 kernel panic
（'completeMemory() prepare count underflow'，见 oMLX #300, #435）。

**修复**: 所有 9 处 `mx.clear_cache()` 调用前添加 `mx.synchronize()`。

**影响文件**: batch_generate.py, runner.py, utils_mlx.py, opt_batch_gen.py

### P0-B: Metal 内存 API 替代 psutil
**问题**: `psutil.virtual_memory()` 包含 OS 缓存，不准确反映 Metal GPU 实际占用。

**修复**: 改用 `mx.get_active_memory()` 直接追踪 Metal 分配量。

**新增环境变量**: `EXO_PREFILL_MAX_METAL_GB`（默认自动：总内存 - 1.5GB）

**影响文件**: generate.py, cache.py

### P0-C: Prefill 峰值内存预估
**问题**: 预填充开始后才发现内存不足，导致 OOM。

**修复**: 新增 `_estimate_prefill_peak_bytes()` 预飞检查，考虑 SDPA head_dim：
- `head_dim > 128`: MLX SDPA 回退全注意力矩阵 `[B, heads, chunk, total_kv_len]` float32
- `head_dim ≤ 128`: MLX 融合 SDPA kernel，峰值很小

在 `prefill()` 开始前检查，如预计峰值超限则提前抛出 `PrefillOOM`。

**影响文件**: generate.py

### P1-D: 延迟 Metal 缓存清理
**问题**: oMLX 发现立即清理可能与 IOKit 异步回调冲突。

**修复**: 任务完成后延迟 8 步（`_DEFERRED_CLEAR_DELAY = 8`）再执行同步清理。

**影响文件**: batch_generate.py

### P1-E: 生成期间内存监控
**修复**: 生成过程中每 64 步检查 Metal 内存并记录日志。

**新增环境变量**: `EXO_GENERATION_MEMORY_CHECK_INTERVAL`（默认 64）

**影响文件**: batch_generate.py

### P2: SSD KV Cache 持久化
**问题**: KV prefix cache 被 LRU 淘汰后数据丢失，再次请求需完全重新预填充。

**修复**: 新增 `SSDKVStore`（`ssd_kv_cache.py`），在 LRU 淘汰前将 KV 缓存序列化到 SSD：
- 支持 `RotatingKVCache`（注意力层）和 `ArraysCache`（GatedDeltaNet 层）
- 使用 safetensors 格式序列化（MLX memory-mapped 加载）
- 进程重启后自动扫描并恢复 SSD 缓存条目
- RAM miss 后自动查找 SSD 作为 fallback
- 初始化失败（如磁盘不可用）自动降级为纯内存模式

**新增环境变量**:
- `EXO_SSD_CACHE_ENABLED`（默认 `1`）
- `EXO_SSD_CACHE_DIR`（默认 `~/.cache/exo/kv_cache`）
- `EXO_SSD_CACHE_MAX_GB`（默认 `10`）

**新增文件**: `src/exo/worker/engines/mlx/ssd_kv_cache.py`
**修改文件**: `src/exo/worker/engines/mlx/cache.py`

## 所有新增环境变量汇总

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `EXO_PREFILL_MAX_METAL_GB` | `0`（自动=总内存-1.5GB）| Metal 内存预填充限制 |
| `EXO_GENERATION_MEMORY_CHECK_INTERVAL` | `64` | 生成时内存检查间隔步数 |
| `EXO_SSD_CACHE_ENABLED` | `1` | 是否启用 SSD KV 缓存 |
| `EXO_SSD_CACHE_DIR` | `~/.cache/exo/kv_cache` | SSD 缓存目录 |
| `EXO_SSD_CACHE_MAX_GB` | `10` | SSD 缓存最大容量(GB) |
