# 2026-04-17 OpenClaw 逐个技能裁剪

## 背景

此前虽然已经恢复了 OpenClaw system prompt 的默认 `3000` 字符裁剪，
但 `<available_skills>` 内部仍主要依赖“描述行截断 + 最终总长兜底”。

这会留下两个问题：

- 某些 skill 的 `<description>` 跨多行时，压缩不够彻底。
- 即使描述被裁短，skill 条目本身仍可能保留松散换行，导致中间段偏长。

## 修复

将 OpenClaw `<available_skills>` 的裁剪逻辑升级为“逐个 `<skill>` 重写”：

1. 逐个解析 `<skill>...</skill>`。
2. 提取并保留：
   - `<name>`
   - `<description>`
   - `<location>`
3. 将 `<description>` 压短到 `EXO_OPENCLAW_SKILLS_DESC_MAX_CHARS`
   或显式配置的 `EXO_SKILLS_DESC_MAX_CHARS`。
4. 将每个 skill 重写为统一紧凑结构，去掉多余换行膨胀。

如果某个 skill 条目结构异常、无法正常解析，则回退到原有的逐行裁剪逻辑，
避免因为格式不规则而直接丢内容。

## 效果

- “中间技能太长”的情况会更早在每个 skill 级别收缩，而不是只靠最后整体截断。
- 跨行 description 会被折叠成单行紧凑格式。
- `name` 与 `location` 继续保留，避免影响后续按 skill 读取 `SKILL.md` 的能力。
